<task type="auto">
  <name>Task 1: Deploy via Ansible and run automated smoke tests</name>
  <files></files>
  <action>
Run the Ansible playbook to deploy the updated modules. This will stow the new conf.d fragment files and the rewritten .zshrc into the home directory.

```bash
ansible-playbook deploy.yml
```

Then run automated smoke tests:

1. **Shell opens without error:**
   ```bash
   zsh -l -c 'echo "shell-ok"'
   ```
   Expected: prints "shell-ok" with no errors.

2. **DOTFILES_DEBUG works (SHRC-04):**
   ```bash
   DOTFILES_DEBUG=1 zsh -l -c '' 2>&1
   ```
   Expected: output includes lines like `sourcing: /Users/mcravey/.zsh/conf.d/10-zsh-core.sh` for each fragment.

3. **conf.d directory has all 4 fragments (MIGR-02, SHRC-03):**
   ```bash
   ls -la ~/.zsh/conf.d/
   ```
   Expected: 4 symlinks with correct numeric prefixes (10, 50, 50, 80).

4. **EDITOR/VISUAL only in editor fragment (MIGR-04):**
   ```bash
   grep -l 'export EDITOR\|export VISUAL' ~/.zsh/conf.d/*.sh
   ```
   Expected: only `50-editor-env-aliases.sh`.

5. **Attribution headers present (MIGR-03):**
   ```bash
   head -2 ~/.zsh/conf.d/*.sh
   ```
   Expected: each file has shellcheck directive and module attribution.

6. **Glob loop present in .zshrc (SHRC-01):**
   ```bash
   grep 'for conf in.*conf.d.*\.sh(N)' ~/.zshrc
   ```
   Expected: matches the conf.d glob loop line.

If ansible-playbook fails due to stale symlinks from old files, run `stow -D` for the affected modules first, then re-run deploy. The old files (environment.sh, aliases.sh, etc.) were removed from the repo, so stow needs to clean up their symlinks.
  </action>
  <verify>
    <automated>zsh -l -c 'echo "shell-ok"' 2>&1 | grep -q "shell-ok" && DOTFILES_DEBUG=1 zsh -l -c '' 2>&1 | grep -q "sourcing:" && ls ~/.zsh/conf.d/*.sh 2>/dev/null | wc -l | tr -d ' ' | grep -q "4" && echo "PASS" || echo "FAIL"</automated>
  </verify>
  <done>Ansible deploy succeeds, zsh opens without errors, DOTFILES_DEBUG prints 4 sourced fragments, conf.d directory has 4 symlinked fragment files</done>
</task>
