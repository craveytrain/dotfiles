- trusted_config_paths in dev-tools.toml covers the entire conf.d directory (TOOL-02)
- asdf_compat=true preserved per user decision

Create the conf.d directories if they don't exist.

Three sub-tasks:

**A) Add MISE_TRUSTED_CONFIG_PATHS to zsh activation script** (`modules/dev-tools/files/.zsh/conf.d/80-dev-tools-mise.sh`):
```bash
# shellcheck shell=zsh
# dev-tools module - mise-en-place runtime manager activation

export MISE_TRUSTED_CONFIG_PATHS="$HOME/.config/mise/conf.d"
eval "$(mise activate zsh)"
```

**B) Add MISE_TRUSTED_CONFIG_PATHS to fish activation script** (`modules/dev-tools/files/.config/fish/conf.d/80-dev-tools-mise.fish`):
```fish
# dev-tools module - mise runtime manager activation

set -gx MISE_TRUSTED_CONFIG_PATHS "$HOME/.config/mise/conf.d"
if status --is-interactive
    mise activate fish | source
end
```

The env var export goes BEFORE the interactive guard and BEFORE `mise activate` so mise sees it during activation. This solves the trust bootstrap chicken-and-egg problem (see RESEARCH.md Pitfall 1).

**C) Remove old files:**
- Delete `modules/dev-tools/files/.config/mise/config.toml` (replaced by conf.d/dev-tools.toml)
- Delete `modules/node/files/.config/mise/config.toml` (replaced by conf.d/node.toml)
- Delete `modules/dev-tools/files/.tool-versions` (stale artifact declaring `nodejs 22.9.0`, conflicts with node module's `latest`)

Do NOT modify `mergeable_files` in config.yml files. That cleanup is Phase 7 (MIGR-01). The mergeable_files entry becomes a no-op once the source config.toml file doesn't exist.
